## 1. 功能目标
提供一个安全、高效的待办事项管理 REST API，允许用户通过 JWT 认证后对自己的待办事项进行创建、查看、标记完成和删除操作，确保用户数据严格隔离，并遵循统一的错误处理规范。

## 2. 用户故事
| 角色 | 行为 | 期望结果 |
|------|------|---------|
| 新用户 | 使用用户名和密码注册 | 成功创建账号，返回用户信息（密码不回传） |
| 已注册用户 | 使用用户名和密码登录 | 获取有效的 JWT Token 及用户基本信息 |
| 已登录用户 | 创建待办事项（标题必填，描述和截止日期可选） | 待办创建成功，返回包含 id 的待办对象 |
| 已登录用户 | 查看自己的待办列表（支持分页） | 收到按创建时间倒序排列的待办列表及分页信息 |
| 已登录用户 | 查看自己的单个待办详情 | 返回该待办的完整信息 |
| 已登录用户 | 标记自己的待办为已完成 | 待办状态变为已完成，返回更新后的对象 |
| 已登录用户 | 删除自己的待办 | 待办被删除，返回 204 |
| 已登录用户 | 尝试访问他人的待办 | 返回 403 Forbidden |
| 未登录用户 | 访问任何待办接口 | 返回 401 Unauthorized |

## 3. 功能点
### 3.1 用户注册
- **HTTP 方法**：POST
- **描述**：用户通过用户名和密码注册新账号。
- **请求体**：JSON（Content-Type: application/json），包含 `username`（string，必填）、`password`（string，必填）。
- **输入校验**：
  - `username`：允许大小写字母、数字、下划线，长度 3-20 个字符；不满足时返回 400。
  - `password`：最小长度 8 位，必须同时包含字母和数字；不满足时返回 400。
- **输出**：成功返回 201 和用户信息 `{ "id": int, "username": string }`；失败返回错误码及错误信息。
- **前置条件**：无。
- **后置条件**：新用户记录创建，密码加密存储（如 bcrypt）。用户名存储为统一小写形式（例如 "User1" 存储为 "user1"）。

### 3.2 用户登录
- **HTTP 方法**：POST
- **描述**：已注册用户通过用户名和密码登录，获取 JWT Token。
- **请求体**：JSON（Content-Type: application/json），包含 `username`（string，必填）、`password`（string，必填）。
- **输出**：成功返回 200 和 `{ "token": string, "user": { "id": int, "username": string } }`；失败返回 401 及错误信息。
- **前置条件**：用户已注册。
- **后置条件**：服务端生成 Token（过期时间 24 小时），客户端需在后续请求的 Authorization 头携带（Bearer Token）。

### 3.3 创建待办事项
- **HTTP 方法**：POST
- **描述**：认证用户创建一条新的待办事项。
- **请求体**：JSON（Content-Type: application/json），包含 `title`（string，必填）、`description`（string，可选）、`due_date`（string，可选，格式 YYYY-MM-DD）。
- **输出**：成功返回 201 和待办对象 `{ "id": int, "title": string, "description": string | null, "due_date": string | null, "completed": false, "created_at": datetime, "updated_at": datetime, "user_id": int }`。
- **前置条件**：用户已认证（请求携带有效 JWT）。
- **后置条件**：新待办记录创建并与当前用户关联。标题不能为空，否则返回 400。

### 3.4 获取待办事项列表
- **HTTP 方法**：GET
- **描述**：认证用户获取自己的待办列表，支持分页，按创建时间倒序。
- **查询参数**：`page`（int，可选，默认 1，最小 1）、`page_size`（int，可选，默认 20，最小 1，最大 100）。
- **输出**：成功返回 200 和 `{ "items": [...], "total": int, "page": int, "page_size": int, "total_pages": int }`。
  - 当列表为空时，`items` 返回空数组 `[]`，分页元数据正常返回。
  - 当 `page` 超过总页数时，`items` 返回空数组 `[]`，`page` 仍为请求值，不返回 400。
- **前置条件**：用户已认证。
- **后置条件**：无。

### 3.5 获取待办事项详情
- **HTTP 方法**：GET
- **描述**：认证用户获取单个待办的详细信息。
- **输入**：路径参数 `todo_id`。
- **输出**：
  - 成功返回 200 和待办对象。
  - 若 `todo_id` 不存在：返回 404。
  - 若 `todo_id` 存在但属于其他用户：返回 403。
- **前置条件**：用户已认证。
- **后置条件**：无。

### 3.6 标记待办事项为已完成
- **HTTP 方法**：PATCH
- **描述**：认证用户将某个自己的待办状态标记为已完成。
- **输入**：路径参数 `todo_id`。
- **输出**：
  - 成功返回 200 和更新后的待办对象（`completed=true`）。
  - 若 `todo_id` 不存在：返回 404。
  - 若 `todo_id` 存在但属于其他用户：返回 403。
- **前置条件**：用户已认证。
- **后置条件**：待办记录更新 `completed` 字段为 `true`。若已经完成，视为成功，仍返回 200 和当前对象。

### 3.7 删除待办事项
- **HTTP 方法**：DELETE
- **描述**：认证用户删除自己的某个待办。
- **输入**：路径参数 `todo_id`。
- **输出**：
  - 成功返回 204 无内容。
  - 若 `todo_id` 不存在：返回 404。
  - 若 `todo_id` 存在但属于其他用户：返回 403。
- **前置条件**：用户已认证。
- **后置条件**：待办记录从数据库删除。

## 4. 业务规则
- 用户名必须唯一，不区分大小写时视为相同；存储时统一转为小写以确保唯一性。
- 用户名允许的字符集：大小写英文字母、数字、下划线；长度 3-20 个字符。
- 密码需经过加密存储（如 bcrypt），且必须满足：最小长度 8 位，同时包含至少一个字母和一个数字。
- JWT Token 中必须包含用户 ID，并设置过期时间为 **24 小时**。
- 所有待办接口（创建、列表、详情、完成、删除）均需认证，通过验证 Authorization 头中的 Bearer Token。
- 用户只能操作自己的待办事项；访问他人待办时返回 403 Forbidden。
- 统一错误响应格式：`{ "code": int, "message": string }`，且 `code` 等于 HTTP 状态码。
- 创建待办时标题不能为空，否则返回 400。
- 分页参数 `page` 最小为 1，`page_size` 最小为 1、最大为 100，超出范围（≤0 或 >100）返回 400。
- 截止日期（若提供）必须严格遵循 `YYYY-MM-DD` 格式，否则返回 400。
- 并发冲突检测：对同一待办的并发修改（如同时完成和删除）使用乐观锁（例如版本号字段 `version`），若版本号不匹配则返回 409，包含 `{ "code": 409, "message": "Resource conflict" }`。
- 性能基线：在单用户最多 1000 条待办、总用户数 1000 的条件下，平均响应时间小于 **200ms**。

## 5. 边界条件
| 场景 | 预期行为 |
|------|---------|
| 注册时用户名已存在 | 返回 409 Conflict |
| 注册时用户名为空、超长（>20）或含非法字符 | 返回 400 Bad Request |
| 注册时密码长度不足 8 位或不含字母/数字 | 返回 400 Bad Request |
| 登录时用户名或密码错误 | 返回 401 Unauthorized |
| 未携带 Token 访问受保护接口 | 返回 401 Unauthorized |
| Token 无效或已过期 | 返回 401 Unauthorized |
| 访问不存在的待办 ID | 返回 404 Not Found |
| 访问他人的待办 | 返回 403 Forbidden |
| 创建待办时标题为空 | 返回 400 Bad Request，message 提示标题必填 |
| 创建待办时截止日期格式非法（非 YYYY-MM-DD） | 返回 400 Bad Request，提示日期格式错误 |
| 分页参数 page 为 0 或负数 | 返回 400 Bad Request，提示 page 必须 ≥1 |
| 分页参数 page_size 为 0 或负数 | 返回 400 Bad Request，提示 page_size 必须 ≥1 |
| 分页参数 page_size 超过 100 | 返回 400 Bad Request，提示 page_size 最大 100 |
| 删除不存在的待办 | 返回 404 Not Found |
| 标记已完成不存在的待办 | 返回 404 Not Found |
| 标记已完成一个已完成的待办 | 视为成功，返回 200，completed 仍为 true |
| 并发修改同一个待办（如同时完成和删除） | 原子性保证，一个成功其余失败，返回 409 Conflict，message 为 "Resource conflict" |
| 请求超页（page 大于总页数） | 返回 200，items 为空数组，其他分页元数据正常 |
| 大量并发请求（符合性能基线场景） | 系统正常运作，平均响应时间 <200ms |

## 6. 验收标准
- [ ] 用户可注册新账号（用户名唯一），注册时满足用户名和密码规则，返回用户信息且密码不回传
- [ ] 用户可登录并获得 JWT Token 及用户基本信息（id, username）
- [ ] 未提供 Token 时访问待办接口返回 401
- [ ] 使用有效 Token 可创建待办（标题必填，描述和截止日期可选），返回 201 和待办对象
- [ ] 创建待办时标题为空返回 400
- [ ] 查看待办列表支持分页参数（page, page_size），默认 page=1、page_size=20，按创建时间倒序，返回分页元数据
- [ ] 列表为空或超页时返回空数组 `[]` 而非错误
- [ ] 查看单个待办详情返回正确信息；不存在返回 404；他人的返回 403
- [ ] 标记自己的待办为已完成，返回 200，completed 变为 true；已完成再次标记仍返回 200
- [ ] 删除自己的待办返回 204；不存在返回 404；他人的返回 403
- [ ] 所有错误响应均为 `{ "code": int, "message": string }`，code 与 HTTP 状态码一致
- [ ] 并发冲突（如同时完成和删除）返回 409 及 "Resource conflict"
- [ ] 性能基线满足：每个用户最多 1000 条待办，总用户 1000，平均响应时间 <200ms

## 7. 不做什么（Out of Scope）
- 用户角色管理（管理员、普通用户等）
- 待办事项的编辑（修改标题/描述/截止日期）
- 待办事项的优先级、标签、分类
- 待办事项搜索功能（仅提供列表分页）
- 待办事项协作/共享
- 退出登录（Token 注销）及 Refresh Token 机制
- 忘记密码 / 重置密码
- 修改用户信息（用户名、密码等）
- 国际化和多语言

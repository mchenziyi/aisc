# 待办事项 API - 需求文档

## 概述
开发一个简单的待办事项（Todo）管理 REST API，支持创建、查看、完成、删除待办事项。

## 核心功能

### 1. 用户认证
- 用户通过用户名+密码注册
- 登录后获取 JWT Token
- 所有待办事项接口需要认证

### 2. 待办事项管理
- 创建待办事项：标题（必填）、描述（可选）、截止日期（可选）
- 查看待办事项列表：支持分页（page, page_size），按创建时间倒序
- 查看单个待办事项详情
- 标记为已完成
- 删除待办事项

### 3. 权限
- 用户只能查看和管理自己的待办事项
- 不能访问他人的待办事项

### 4. 错误处理
- 统一的错误响应格式：{ code, message }
- 常见错误码：400, 401, 403, 404, 500

## 非功能性需求
- 响应时间 < 200ms
- 支持 10,000 并发用户

## 1. 功能目标
构建一个专注人工智能领域的视频分享平台，使注册用户能够上传、搜索、浏览和收藏 AI 相关视频，从而促进 AI 知识与经验的传播和交流。

## 2. 用户故事
| 角色 | 行为 | 期望结果 |
|------|------|---------|
| 注册用户 | 填写标题、描述和标签并上传视频文件 | 视频成功保存并能在视频列表中检索到，其他用户可以观看 |
| 注册用户 | 在视频列表页使用关键词搜索或翻页浏览 | 得到匹配的视频结果列表，支持分页导航 |
| 注册用户 | 在列表页或详情页点击“收藏”按钮 | 视频被加入个人收藏夹，按钮状态切换为已收藏 |
| 注册用户 | 进入“我的收藏”页面 | 看到所有收藏过的视频，支持删除收藏 |
| 注册用户 | 进入“我上传的”页面 | 查看自己上传的所有视频，并可编辑或删除 |

## 3. 功能点
### 3.1 用户注册与登录
- 描述：用户通过邮箱/用户名和密码注册账号，注册后登录系统；未登录用户无法访问任何业务功能（仅能看到登录/注册页面）。
- 输入：注册：用户名、邮箱、密码；登录：用户名/邮箱、密码。
- 输出：注册成功后自动登录并跳转至视频列表页；登录成功后返回用户会话 token（JWT）。
- 前置条件：无
- 后置条件：用户已认证，可以执行上传、浏览、收藏等操作。

### 3.2 视频上传
- 描述：用户上传 mp4 视频文件，并填写标题（必填）、描述（选填）、标签（选填，最多 5 个）。采用预签名直传机制：客户端先通过 POST /uploads/token 获取上传凭证，再直接上传至对象存储；上传完成后客户端必须调用服务端创建视频接口（POST /videos）提交元数据，视频进入公开状态。
- 上传凭证获取接口（POST /uploads/token）：
  - 请求参数（JSON Body）：
    - filename（string）：文件名。
    - fileSize（number）：文件大小，单位为字节。
    - mimeType（string）：文件 MIME 类型，如 "video/mp4"。
  - 响应（JSON）：
    - uploadUrl（string）：临时上传地址（如预签名 URL 或表单上传地址）。
    - fields（object）：上传所需的凭证字段，可包含 key、policy、signature 等（具体字段依赖对象存储实现）。
    - uploadId（string）：上传标识，服务端内部关联本次上传操作，用于后续视频创建时定位已上传文件。
- 全链路流程：
  1. 客户端填写标题、描述、标签，并选择视频文件。
  2. 客户端调用 POST /uploads/token，传入 filename、fileSize、mimeType，获取上传凭证（uploadUrl、fields、uploadId）。
  3. 客户端使用凭证中的 uploadUrl 和 fields，将视频文件直接上传至对象存储。
  4. 上传成功后，客户端调用 POST /videos，提交 title（必填）、description（选填）、tags（选填，逗号分隔）以及 uploadId（必填，取自上传凭证响应）。
  5. 服务端根据 uploadId 定位已上传的视频文件，通过 FFmpeg 解析文件，强制校验时长（≤10 分钟）和编码格式（H.264）。校验通过后，创建视频记录，生成视频 ID，并返回给客户端；若校验失败，返回错误详情并拒绝创建。
  6. 视频即刻进入公开状态。
- 输入：
  - 获取凭证阶段：文件名、文件大小、文件类型。
  - 创建视频阶段：标题（必填）、描述（选填）、标签（选填）、uploadId（必填）。
- 输出：最终返回视频 ID（字符串）。
- 前置条件：用户已登录。
- 后置条件：视频出现在公开视频列表中，用户可在“我上传的”页面管理。

### 3.3 视频浏览与搜索
- 描述：用户通过视频列表页查看所有已发布的视频，支持按标题、描述、标签进行关键词搜索，结果分页展示。分页查询参数和响应元数据格式统一约定如下：
  - 查询参数：keyword（可选，最长 100 字符）、page（整数，默认 1）、page_size（整数，默认 20，最小 1，最大 100）。
  - 响应元数据：total（总记录数）、page（当前页）、page_size（当前页大小）。
  - 列表数据置于 data 字段中。
- 输入：keyword（query 参数，可选）、page（query 参数，默认 1）、page_size（query 参数，默认 20，范围 1-100）。
- 输出：{
    data: [视频摘要对象],
    total: number,
    page: number,
    page_size: number
  }
  其中视频摘要对象包含：视频 ID、标题、上传者用户名、封面 URL（一期使用统一默认封面）、收藏数、上传时间、当前用户是否已收藏。
- 前置条件：用户已登录。
- 后置条件：无。
- 分页约定：本接口及以下 3.4、3.6 中所有分页接口均采用相同查询参数名（page、page_size）和响应元数据格式（total、page、page_size）。

### 3.4 收藏管理
- 描述：用户可以在视频列表页和视频详情页对某个视频进行收藏或取消收藏（通过 POST /videos/{id}/favorite toggle 接口）。用户可通过 GET /favorites 接口获取当前用户收藏的视频列表（支持分页，分页参数同 3.3 约定），在“我的收藏”页面展示。取消收藏同样调用 toggle 接口。
- 输入：获取列表：分页参数 page、page_size（同 3.3 约定）；收藏操作：视频 ID（路径参数）。
- 输出：获取列表：响应格式同 3.3（data 为视频摘要列表，仅包含已收藏的视频；分页元数据 total、page、page_size）；收藏操作：最新收藏状态（true/false）及该视频最新收藏数。
- 前置条件：用户已登录，视频存在且已发布（仅收藏操作）。
- 后置条件：收藏状态变更，用户收藏列表更新。

### 3.5 视频详情与播放
- 描述：用户点击视频进入详情/播放页面。页面展示视频播放器（获得带鉴权的临时播放 URL）、标题、描述、标签、上传者、上传时间、收藏数、当前用户收藏状态。同时提供收藏按钮。数据通过 GET /videos/{id} 接口获取。
- 输入：视频 ID（路径参数）。
- 输出：视频播放 URL（预签名，有效期可配置）、标题、描述、标签、上传者（用户名）、上传时间、收藏数、当前用户收藏状态。
- 前置条件：用户已登录。
- 后置条件：用户可进行收藏操作。

### 3.6 我上传的视频管理
- 描述：用户可通过 GET /videos?owner=me 接口查看自己上传的所有视频列表（支持分页，分页参数同 3.3 约定），并可编辑视频的标题、描述、标签（通过 PUT /videos/{id}），以及删除视频（通过 DELETE /videos/{id}）。删除操作将永久移除视频文件。
- 输入：查看：分页参数 page、page_size（同 3.3 约定）；编辑：视频 ID（路径参数）、新标题/描述/标签（JSON Body）；删除：视频 ID（路径参数）。
- 输出：查看：响应格式同 3.3（data 为视频摘要列表，仅包含自己上传的视频；分页元数据 total、page、page_size）；编辑：更新成功；删除：删除成功。
- 前置条件：用户已登录，操作仅作用户自己的视频。
- 后置条件：视频信息变更或移除。

## 4. 业务规则
- 4.1 用户必须登录后才能进行任何操作（包括查看视频列表），未登录用户只能看到登录/注册界面。
- 4.2 视频文件格式仅支持 mp4（H.264 编码），其他格式上传时提示并拒绝。单个文件大小不超过 500MB；时长不超过 10 分钟。服务端在 POST /videos 接口中通过 FFmpeg 解析视频文件进行强制校验（时长、编码格式），不符合要求的请求返回错误详情。
- 4.3 标题必填，长度 1~100 字符；描述限 500 字符；标签最多 5 个，每个标签限 20 字符。
- 4.4 视频上传后默认状态为“公开”，一期不提供管理员下架功能，视频状态仅“公开”，无“下架”状态。
- 4.5 搜索基于标题、描述和标签模糊匹配，按上传时间倒序排列。
- 4.6 每个用户对同一视频只能收藏一次，重复点击交替切换收藏/取消收藏（toggle 设计）。
- 4.7 用户只能查看和操作自己的收藏，不能查看他人收藏列表。
- 4.8 视频文件存储于私有对象存储，播放时由服务生成临时预签名 URL（默认有效期 1 小时），客户端不可直接访问永久存储地址。
- 4.9 登录安全：密码长度至少 6 位，建议包含字母和数字。认证使用 JWT，Token 有效期 24 小时，过期需重新登录。一期不实现邮箱验证及 Token 刷新。
- 4.10 搜索关键词最长 100 字符，前后端均需对特殊字符进行安全转义以防止注入。

## 5. 边界条件
| 场景 | 预期行为 |
|------|---------|
| 空数据 | 视频列表页显示“暂无视频，快去上传吧”；搜索无结果显示“未找到相关视频” |
| 极限值 | 上传视频大小恰好 500MB / 时长恰好 10 分钟：允许上传；超过时清晰报错；标题 100 字符正常保存，101 字符提示超限 |
| 并发 | 同一用户快速双击收藏：应保证只产生一条收藏记录（后端通过唯一索引或锁保证幂等） |
| 异常输入 | 上传非 mp4 文件（如 .exe、.mov）：拒绝并提示“仅支持 mp4 格式”；搜索包含特殊字符：安全转义后正常搜索 |
| 网络异常 | 上传中断：提示用户重新上传，不支持断点续传；列表/详情加载失败：显示错误提示及重试按钮 |
| 分页越界 | 请求页码超出总页数：返回空列表（data: []），分页元数据（total、page、page_size）正常返回，page 为请求的页码，total 为实际总记录数 |

## 6. 验收标准
- [ ] 用户可注册新账号，并使用该账号登录系统；未登录访问任何页面被重定向至登录页。
- [ ] 登录后，用户可以在上传页选择 mp4 视频文件、填写标题/描述/标签，点击上传后视频成功显示在视频列表中。
- [ ] 视频列表页展示所有公开视频，按上传时间降序排列，支持输入关键词搜索并显示匹配结果。
- [ ] 列表支持分页，每页显示固定数量，翻页时正确更新内容；请求超出最大页数时返回空列表（含正确分页元数据）。
- [ ] 用户点击收藏按钮后，按钮变为已收藏状态，收藏数 +1；再次点击恢复原状态，收藏数 -1（列表和详情页均可用）。
- [ ] 用户进入“我的收藏”页面，只能看到自己收藏的视频，支持取消收藏并从列表消失。
- [ ] 用户进入“我上传的”页面，可查看自己所有视频，并支持编辑标题/描述/标签以及删除视频。
- [ ] 视频文件大小、格式、时长、标题长度等限制按规则生效，超限时前后端给出清晰错误提示。
- [ ] 视频详情页展示播放器、标题、描述、标签、上传者、上传时间、收藏数和收藏状态；播放 URL 签名正确且仅在登录状态下有效。
- [ ] 不支持未登录用户执行任何业务操作（全部跳转至登录页）。

## 7. 不做什么（Out of Scope）
- 视频转码/自适应码率（第一阶段直接上传原文件，播放时用 HTML5 原生播放器）
- 用户间的关注/私信/评论功能
- AI 自动生成视频内容、字幕、缩略图等（后续迭代）
- 管理员后台管理（视频审核、用户管理等）
- 视频下载权限控制（允许用户下载原文件，但不做限制）
- 第三方登录（微信、GitHub 等）
- 移动端 App（仅 Web 端，后续迭代）
- 断点续传
- Token 刷新机制

## 1. 功能目标
构建一个专注人工智能领域的视频分享平台，使注册用户能够上传、搜索、浏览和收藏 AI 相关视频，从而促进 AI 知识与经验的传播和交流。

## 2. 用户故事
| 角色 | 行为 | 期望结果 |
|------|------|---------|
| 注册用户 | 填写标题、描述和标签并上传视频文件 | 视频成功保存并能在视频列表中检索到，其他用户可以观看 |
| 注册用户 | 在视频列表页使用关键词搜索或翻页浏览 | 得到匹配的视频结果列表，支持分页导航 |
| 注册用户 | 浏览视频时点击“收藏”按钮 | 视频被加入个人收藏夹，按钮状态切换为已收藏 |
| 注册用户 | 进入“我的收藏”页面 | 看到所有收藏过的视频，支持删除收藏 |

## 3. 功能点
### 3.1 用户注册与登录
- 描述：用户通过邮箱/用户名和密码注册账号，注册后登录系统；未登录用户无法访问任何业务功能（仅能看到登录/注册页面）。
- 输入：注册：用户名、邮箱、密码；登录：用户名/邮箱、密码。
- 输出：注册成功后自动登录并跳转至视频列表页；登录成功后返回用户会话 token 或 session。
- 前置条件：无
- 后置条件：用户已认证，可以执行上传、浏览、收藏等操作。

### 3.2 视频上传
- 描述：用户上传视频文件，并填写标题（必填）、描述（选填）、标签（选填，最多 5 个）。
- 输入：视频文件（格式与大小见业务规则）、标题、描述、标签（逗号分隔）。
- 输出：视频上传成功，系统返回视频 ID，并触发转码（如有），视频进入待审核或立即发布状态（根据规则）。
- 前置条件：用户已登录。
- 后置条件：视频出现在视频列表中（若状态为已发布），用户可在“我上传的”页面管理。

### 3.3 视频浏览与搜索
- 描述：用户通过视频列表页查看所有已发布的视频，支持按照标题/描述/标签进行关键词搜索，结果分页展示。
- 输入：搜索关键词（可选）、页码（默认第1页）、每页条数（默认 20）。
- 输出：视频摘要列表（封面、标题、上传者、收藏数、上传时间），总记录数、总页数。
- 前置条件：用户已登录。
- 后置条件：无

### 3.4 收藏管理
- 描述：用户可以对某个视频进行收藏或取消收藏；用户可以在“我的收藏”页面查看所有收藏的视频，并可删除（取消收藏）。
- 输入：点击收藏或取消收藏按钮，传递视频 ID。
- 输出：收藏/取消收藏成功，收藏数更新。
- 前置条件：用户已登录，视频存在且已发布。
- 后置条件：收藏状态变更，用户收藏列表更新。

## 4. 业务规则
- 4.1 用户必须登录后才能进行任何操作（包括查看视频列表），未登录用户只能看到登录/注册界面。
- 4.2 视频文件格式支持 mp4、mov、avi；单个文件大小不超过 500MB；时长建议 10 分钟以内（可配置）。
- 4.3 标题必填，长度 1~100 字符；描述限 500 字符；标签最多 5 个，每个标签限 20 字符。
- 4.4 视频上传后默认状态为“公开”，管理员可下架（第一阶段不做人工审核，直接发布）。
- 4.5 搜索基于标题和标签模糊匹配，按上传时间倒序排列。
- 4.6 每个用户对同一视频只能收藏一次，重复点击交替切换收藏/取消收藏。
- 4.7 用户只能查看和操作自己的收藏，不能查看他人收藏列表。

## 5. 边界条件
| 场景 | 预期行为 |
|------|---------|
| 空数据 | 视频列表页显示“暂无视频，快去上传吧”；搜索无结果显示“未找到相关视频” |
| 极限值 | 上传视频大小恰好 500MB / 时长较长：系统应校验并给出明确提示；标题 100 字符正常保存，101 字符提示超限 |
| 并发 | 同一用户快速双击收藏：应保证只产生一条收藏记录（幂等处理） |
| 异常输入 | 上传非视频文件（如 .exe）：拒绝并提示“仅支持视频格式”；搜索包含特殊字符：安全转义后正常搜索 |
| 网络异常 | 上传中断：前端可断点续传（如不实现则提示用户重试）；列表加载失败：显示错误提示及重试按钮 |

## 6. 验收标准
- [ ] 用户可注册新账号，并使用该账号登录系统；未登录访问任何页面被重定向至登录页。
- [ ] 登录后，用户可以在上传页选择视频文件、填写标题/描述/标签，点击上传后视频成功显示在视频列表中。
- [ ] 视频列表页展示所有公开视频，按上传时间降序排列，支持输入关键词搜索并显示匹配结果。
- [ ] 列表支持分页，每页显示固定数量，翻页时正确更新内容。
- [ ] 用户点击收藏按钮后，按钮变为已收藏状态，收藏数 +1；再次点击恢复原状态，收藏数 -1。
- [ ] 用户进入“我的收藏”页面，只能看到自己收藏的视频，支持取消收藏并从列表消失。
- [ ] 视频文件大小、格式、标题长度等限制按规则生效，超限时前端和后端均给出清晰错误提示。
- [ ] 不支持未登录用户执行上传、浏览、收藏等操作（全部跳转至登录页）。

## 7. 不做什么（Out of Scope）
- 视频转码/自适应码率（第一阶段直接上传原文件，播放时用 HTML5 原生播放器）
- 用户间的关注/私信/评论功能
- AI 自动生成视频内容、字幕、缩略图等（后续迭代）
- 管理员后台管理（视频审核、用户管理等）
- 视频下载权限控制（允许用户下载原文件，但不做限制）
- 第三方登录（微信、GitHub 等）
- 移动端 App（仅 Web 端，非响应式？移动端后续）

1. 平台名称“AI”具体指什么？是仅允许 AI 相关内容，还是利用 AI 技术辅助平台功能（如推荐、标签智能提取）？需要明确以便调整边界。

4. 用户注册是否需要邮箱验证？当前未包含，需确认安全性等级。
5. 对于“智能”搜索（如语义搜索、个性化推荐）是否属于核心功能？当前只做基础模糊匹配。
6. 是否需要记录视频观看历史或个人播放列表？当前只做收藏。
